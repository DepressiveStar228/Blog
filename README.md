# 📝 Spring Boot Blog Platform

Проста, але багатофункціональна платформа для ведення блогу, розроблена на базі **Java** та **Spring Boot**. Проєкт реалізує повноцінну взаємодію користувачів: від базової реєстрації та відновлення паролів до управління власним контентом та коментування записів.

## 🚀 Технологічний стек

* **Backend:** Java, Spring Boot (Web, Security, Data JPA, Validation, Mail)
* **Frontend:** HTML, Thymeleaf (Server-Side Rendering)
* **База даних:** H2 In-Memory Database (швидкий старт без додаткових налаштувань)
* **Тестування:** BDD сценарії (Cucumber)

---

## ✨ Основні можливості (Features)

### 🔐 Аутентифікація та Безпека
* **Реєстрація:** Валідація даних (перевірка унікальності email та username, пароль не менше 6 символів).
* **Авторизація:** Безпечний вхід та вихід із системи через Spring Security.
* **Відновлення пароля:** Скидання пароля через унікальний токен, який генерується та надсилається користувачу.
* **Доступ:** Гості можуть переглядати пости, але для створення контенту чи коментування потрібна авторизація.

### ✍️ Управління публікаціями (Пости)
* **Повний цикл (CRUD):** Створення, перегляд, редагування та видалення публікацій.
* **Захист авторства:** Редагувати або видаляти пост може тільки той користувач, який його створив.
* **Стрічка:** Відображення всіх постів у зворотному хронологічному порядку.

### 💬 Система коментарів
* **Взаємодія:** Можливість залишати коментарі під будь-яким постом для зареєстрованих користувачів.
* **Модерація:** Користувач може видалити свій коментар.
* **Автор поста** має право видалити будь-який коментар під своєю публікацією.

### 👤 Профілі користувачів
* **Особистий кабінет:** Перегляд власної активності та редагування біографії чи імені користувача.
* **Публічні профілі:** Можливість переглядати сторінки інших авторів, їхні біо та список їхніх постів.

## 🛠️ Встановлення та запуск

1.  **Клонуйте репозиторій:**
    ```bash
    git clone <url_вашого_репозиторію>
    cd blog-platform
    ```

2.  **Збірка та запуск (через Maven):**
    ```bash
    ./mvnw spring-boot:run
    ```
    *Або запустіть проєкт через вашу IDE (IntelliJ IDEA, Eclipse).*

3.  **Доступ до додатку:**
    Відкрийте [http://localhost:8080](http://localhost:8080) у вашому браузері.

4.  **Консоль бази даних H2:**
    Доступна за адресою `http://localhost:8080/h2-console`.

## 📂 Структура проєкту
* `controller/` — обробка вхідних HTTP запитів та навігація.
* `service/` — реалізація бізнес-логіки додатка.
* `config/` — налаштування безпеки та доступу.
* `dto/` — об'єкти для передачі даних між шарами.
* `entity` — об'єкти сутності бази даних.
* `repository` — репозиторії для роботи з об'єктами сутностями.
* `features/` — BDD специфікації функціональності.
